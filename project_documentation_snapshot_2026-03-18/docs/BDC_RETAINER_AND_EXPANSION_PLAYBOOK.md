# BDC Retainer And Expansion Playbook

## Objective
Convert one-off paid pilots into repeat revenue via retainer, repeat redesign, or workflow expansion.

## Offer Paths
- `RP-001`: monthly_architecture_retainer for sales_automation (>=1 paid pilot + proof_ready)
- `RP-002`: adjacent_workflow_expansion for sales_automation (pilot_success_on_first_workflow)
- `RP-003`: valuation_governance_retainer for real_estate (pilot_success_with_review_dependency)
- `RP-004`: multi_workflow_expansion for real_estate (pilot_success_on_matching_or_intake)

## Operating Rules
- Only offer retainers after measured pilot success.
- Use workflow expansion when the first workflow proves value but adjacent workflows remain manual.
- Keep scope and claims inside Restricted BDC boundaries.
