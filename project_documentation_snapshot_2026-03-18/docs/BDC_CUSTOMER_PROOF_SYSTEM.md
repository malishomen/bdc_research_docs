# BDC Customer Proof System

## Objective
Systematically convert successful pilot outcomes into reusable proof assets and testimonials without overclaiming.

## Proof Asset Types
- Metric card: baseline vs BDC vs hybrid numeric deltas.
- Mini-case: context, intervention, outcome, constraints.
- Operator quote: implementation and usability perspective.
- Executive quote: business and decision confidence perspective.

## Testimonial Capture Flow
1. Validate pilot outcomes against agreed criteria.
2. Request consent and publication scope.
3. Draft quote options aligned with measured facts.
4. Customer review and approval.
5. Publish approved asset with claim-safe framing.

## Reuse Rules
- Sales: outcome card + mini-case.
- Outreach: short quote + one quantified result.
- Landing page: flagship metric + scope caveat.
- Review defense: full evidence trail and non-claims.
