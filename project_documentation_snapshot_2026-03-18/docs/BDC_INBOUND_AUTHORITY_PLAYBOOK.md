# BDC Inbound Authority Playbook

## Goal
Convert validated proof assets into repeated inbound trust signals and higher-quality pilot leads.

## Content Loop
1. Select proof asset from latest batch.
2. Convert into one technical + one business-facing format.
3. Publish with explicit restricted-scope framing.
4. Route inbound responses to qualification flow.
5. Feed conversion outcomes back into next content cycle.

## Planned Content Assets
- `PA-001` -> technical_breakdown (blog, cadence=biweekly)
- `PA-002` -> business_case_snapshot (linkedin, cadence=biweekly)
- `PA-003` -> technical_breakdown (webinar, cadence=biweekly)
- `PA-004` -> business_case_snapshot (case_page, cadence=monthly)
- `PA-005` -> technical_breakdown (newsletter, cadence=monthly)
- `PA-006` -> business_case_snapshot (blog, cadence=monthly)

## Scope Discipline
- Always include what BDC does not claim.
- Include negative-result boundaries where relevant.
- Keep proof tied to measurable artifacts.
