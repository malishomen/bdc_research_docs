## 1. Purpose

This document defines the next honest scientific entry back into the original BDC research line after the stabilization of `BDC Designer`.

It does **not** reopen the old `hidden_rule` product path.

It defines how to resume the scientific program without repeating the three major historical errors:
- treating a laboratory benchmark as a product path,
- allowing broken selection physics to masquerade as biology,
- and making universal claims before bounded mechanisms are actually validated.

---

## 2. Starting Position

Current reality:

### What is already operational
- `BDC Designer` is the current productized line.
- It is the mainline-ready subsystem.
- It already has client packet intake, redesign guidance, and operator workflow support.

### What is not reopened automatically
- `hidden_rule` as a direct product trajectory
- the old linear complexity-penalty regime
- universal transition-law ambition

### What remains scientifically alive
- quaternary uncertainty handling
- DNA-inspired memory architecture
- resource-bounded adaptive digital organisms
- architectural specialization and cooperation
- transfer between environments

## 3. Reboot Principle

The research reboot must treat `BDC Designer` and the original scientific line as two coupled but distinct tracks.

### Track A — Product / cash / evidence line
`BDC Designer`

Role:
- sustain operational value,
- bring real workflow evidence,
- finance and discipline further research,
- provide architecture-analysis tools that may later assist research design.

### Track B — Scientific reboot line
`BDC Research`

Role:
- resume the original digital-biology and architecture-emergence program,
- but only through a new validated physics and task setup,
- not by reviving the exhausted `hidden_rule` line.

---

## 4. Hard Lessons That Must Become Rules

## Rule 1 — No more product claims from laboratory benchmarks

`EDP1` remains a lab stand unless a phase explicitly proves external utility.

Implication:
- benchmark wins are not product wins,
- elegant dynamics are not commercial proof.

## Rule 2 — Selection physics comes before organism complexity

The old failure was not “insufficient tuning”.
It was a broken law of selection.

Implication:
- no new ambitious organism architecture until the new fitness/resource regime is validated,
- no “SuperCell” construction by aspiration.

## Rule 3 — One validated layer at a time

The original roadmap already hinted at this, but reboot must enforce it harder:
- no multi-component organism before individual mechanisms pass their own gates.

## Rule 4 — Distinguish biology-inspired mechanism from metaphor

Terms like DNA, cell, membrane, regeneration, metabolism are allowed only when linked to:
- formal mechanism,
- measurement,
- kill criteria,
- and exact implementation semantics.

## Rule 5 — Restricted claims by default

Every phase must start restricted and widen only by evidence.

---

## 5. What The Reboot Is Actually Trying To Answer

The reboot should not ask:
- “Can we finally make the old system work?”

It should ask:
- “What is the smallest scientifically valid path from the original BDC principles to a new adaptive organism line that does not repeat the old regime failure?”

That breaks into four reboot questions:

1. Can we define a biologically credible selection/resource regime that permits complexity growth?
2. Can we define a task/environment that is more meaningful than `hidden_rule` but still controlled enough for causal interpretation?
3. Can we validate one minimal organism component at a time under the new regime?
4. Can we reconnect these components into a genuine adaptive cell line only after those gates pass?

---

## 6. Reboot Architecture

## Layer 0 — Governance reset

Before any new scientific run:
- freeze the old line as historical baseline,
- explicitly define the new reboot line,
- name what is carried forward and what is closed.

Required carry-forward:
- determinism
- reproducibility contracts
- append-only governance
- explicit kill criteria
- negative-result preservation

## Layer 1 — Physics reset

Primary objective:
- define and validate a new resource/energy/complexity regime that does not structurally kill architectural growth.

This must be the first real scientific gate.

No new “organism” claims before this passes.

## Layer 2 — Task reset

Primary objective:
- define a new task family that is richer than `hidden_rule` but still mechanistically interpretable.

Candidate properties:
- not pure toy boolean classification,
- not full messy language benchmarking too early,
- allows causal decomposition,
- supports uncertainty and memory pressure,
- allows observable adaptation.

## Layer 3 — Minimal cell components

Validate separately:
- uncertainty controller
- resource budget / energy mechanism
- memory subsystem
- replication / repair logic
- communication primitive

Only one or two of these should be active in an experiment at a time.

## Layer 4 — Minimal organism assembly

Only after validated components exist:
- combine them into a minimal cell candidate,
- run bounded adaptation tests,
- test stability before sophistication.

## Layer 5 — Cooperation and transfer

Only after a minimal viable cell exists:
- specialization,
- cooperation,
- transfer,
- and panspermia-like claims become legitimate again.

## 7. Proposed Reboot Phases

Execution order is strict.

## Phase R0 — Historical Closure Package

Purpose:
- formalize the historical baseline for the reboot.

Deliverables:
- explicit reboot charter
- frozen list of what is preserved from `EDP1`
- frozen list of what is closed
- causal relation to `BDC Designer`

Definition of Done:
- reboot starts from explicit historical truth, not memory or intuition.

## Phase R1 — Selection Physics Rebuild

Purpose:
- validate new resource/energy-based selection physics.

Required outputs:
- formal regime candidates,
- explicit comparison to old regime,
- proof that complexity growth is mathematically feasible,
- empirical non-collapse under N-seed testing.

Definition of Done:
- at least one new regime passes feasibility and bounded empirical competitiveness criteria.

Failure condition:
- if no regime works, do not advance to “organism” construction.

## Phase R2 — New Controlled Task Environment

Purpose:
- introduce a task/environment that is more meaningful than `hidden_rule` and still scientifically interpretable.

Requirements:
- deterministic,
- causal structure visible,
- supports memory or uncertainty pressure,
- reusable for future components.

Definition of Done:
- task approved as canonical reboot environment with baselines and kill criteria.

## Phase R3 — Minimal Mechanism Validation

Purpose:
- validate one mechanism at a time.

Candidate subtracks:
- quaternary uncertainty advantage,
- indexed memory integrity,
- bounded resource economy,
- repair/recovery protocol,
- communication primitive.

Definition of Done:
- at least one mechanism is validated under the new regime with explicit PASS/FAIL thresholds.

## Phase R4 — Minimal Cell v0

Purpose:
- assemble the smallest organism that is justified by passed components.

Constraints:
- no 9-component fantasy build,
- no feature inflation,
- no aesthetic biomimicry without necessity.

Definition of Done:
- minimal cell survives and adapts under formal test conditions.

## Phase R5 — Cooperation and Transfer

Purpose:
- test whether specialization and transfer can emerge under the new line.

Definition of Done:
- cooperative advantage and transfer must be separately measured and not conflated.

## 8. What Must Stay Product-Separated

The following must not be mixed into the scientific reboot:
- client packet intake tasks,
- pilot metrics,
- commercial GTM logic,
- proof-asset generation as scientific evidence,
- customer workflow redesign outputs as organism research results.

`BDC Designer` can support the research line operationally, but it must not replace actual experiments.

---

## 9. Where `BDC Designer` Can Help The Reboot

`BDC Designer` is useful to the reboot in limited ways:

1. architecture framing for cooperative workflow logic,
2. evidence packet discipline,
3. explicit confidence and insufficiency handling,
4. measurement gap reasoning,
5. design memo generation for research planning.

It is not the source of truth for biological validation.

The reboot plan should be considered correctly launched only if:

1. the old line is explicitly frozen as historical baseline,
2. the new line has a named hypothesis set,
3. the first scientific gate is selection physics, not organism complexity,
4. all claims remain restricted until new evidence widens them,
5. `BDC Designer` remains a separate operational line rather than swallowing the research track.
