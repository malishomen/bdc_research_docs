# BDC Pilot Deployment Program

## Program Goal
Run BDC Designer pilots through a standardized process with clear intake, qualification, execution, and success gates.

## 1) Intake
- Collect workflow descriptor snapshot.
- Confirm baseline measurability and stakeholder owner.
- Confirm scope fit with Restricted BDC boundaries.

## 2) Qualification
- Score lead against qualification matrix.
- Require baseline availability and architecture sensitivity.
- Reject non-fit cases requiring universal guarantees.

## 3) Pilot Execution
1. Freeze baseline.
2. Run CLI recommendation.
3. Apply hybrid refinement.
4. Compare baseline vs BDC vs hybrid.
5. Record operator notes and caution handling.

## 4) Success Criteria
- Time-to-target improvement.
- Quality improvement.
- Search-cost or iteration reduction.
- Documentation completeness for reproducibility.

## 5) Operating Rhythm
- Weekly execution review.
- Bi-weekly stakeholder checkpoint.
- End-of-pilot decision: extend / close / convert.

## Scope and Trust Posture
This program uses Restricted BDC v1 and does not claim universal optimization.
