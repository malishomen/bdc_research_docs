# BDC Early Adopter Conversion Playbook

## Objective
Convert proof assets (flagship + real-task suite + casebook) into a repeatable path from outreach to demo to qualified paid pilot.

## Conversion Funnel
1. ICP qualification
2. Outreach + proof asset share
3. Demo call with workflow-fit check
4. Pilot qualification score
5. Paid pilot proposal
6. Pilot kickoff

## Demo-to-Pilot Rules
- Do not move to pilot without baseline freeze commitment.
- Require decision owner and timeline.
- Require acceptance of restricted BDC scope/non-claims.

## Objection Handling
- Universal optimizer request -> clarify restricted scope and non-claims.
- "No proof" -> show flagship + real suite + casebook pattern matrix.
- "Too expensive" -> offer staged starter pilot with fixed deliverables.

## Priority Contact Logic
Prioritize leads with high architecture sensitivity, baseline readiness, and strong stakeholder ownership.
